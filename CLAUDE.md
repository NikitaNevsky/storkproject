# Stork — продуктовая доставка

Этот файл автоматически читается Claude Code в начале каждой сессии. Содержит ключевой контекст проекта, ссылки на спецификации и правила работы.

## Что это за проект

**Stork (Аист)** — гипер-локальный сервис доставки свежих продуктов и собственной выпечки. Российский клиент, владелец сети ТЦ, запускает продуктовую сеть с пекарней и кухней на каждой точке. Доставка курьерами за 30 минут + самовывоз. Модель похожа на «Самокат», но со слотовой доставкой (как Сбермаркет) и собственным production.

**Запуск:** Москва, Хамовники → весь ЦАО → МО → другие города.

**Языки общения:**
- С клиентом и заказчиком — **только русский**
- Код, коммиты, технические комментарии в коде — на английском
- Документация и README — русский с английскими техническими терминами

## Где спецификация и дизайн

- `docs/STORK-SPEC.md` — полная техническая спецификация (палитра, стэк, архитектура, фазы)
- `design/prototypes/*.html` — финальные HTML-прототипы, согласованные с клиентом (52 экрана мобильного приложения + 9 разделов админки + 8 экранов курьера + 6 страниц веб-витрины)
- `design/prototypes/stork-index.html` — карта проекта со ссылками на все экраны
- `design/photos/` — фотографии товаров и иллюстрации (22 файла)

**Правило:** HTML-прототипы — это design source of truth. Не упрощать визуал без согласования. При создании любого экрана сначала открыть соответствующий HTML, посмотреть как именно он должен выглядеть и работать.

## Стэк (зафиксирован)

- **Mobile:** Expo SDK 51+ / RN / TypeScript / Reanimated 3 / Expo Router / Zustand / TanStack Query / RHF+Zod / MMKV
- **Web/Admin:** Next.js 15 App Router / Tailwind / shadcn/ui / TanStack Table / Yandex Maps
- **Backend:** NestJS / TypeScript strict / PostgreSQL 16 + PostGIS / Prisma / Redis / BullMQ / Socket.io / S3-совместимое хранилище (MinIO в dev)
- **Платежи РФ:** СБП через Тинькофф или CloudPayments + SberPay + карты
- **Инфра:** Docker Compose в dev, VPS с docker-compose в продакшене первого этапа

## Правила работы

1. **TypeScript strict mode** на всех уровнях. Никаких `any`.
2. **Mobile-first** во всём. Десктоп — вторичен.
3. **Один источник дизайн-токенов** — `/packages/ui-tokens`, импортируется и в Tailwind, и в RN StyleSheet.
4. **Один источник DTO** — `/packages/shared-types`, импортируется и фронтом, и бэком.
5. **Async через TanStack Query** на фронте, никаких голых `useEffect + fetch`.
6. **Оптимистические апдейты в корзине** (мгновенный UI-отклик, фоновая синхронизация).
7. **Никаких console.log в проде** — используем Pino на бэке, тонкую обёртку на фронте.
8. **Production-grade с самого начала:** Sentry, structured logging, health-checks, graceful shutdown.
9. **Критичные тесты обязательны:** оформление заказа (e2e), расчёт корзины с бонусами (unit), геозоны (unit).
10. **Каждая spornaya UX-развилка — на согласование с заказчиком**, не догадываемся. Заказчик отвечает на русском, в чате.

1. Прочитать `docs/STORK-SPEC.md` целиком — там детали по архитектуре, моделям данных, API, фазам.
2. Если что-то непонятно или есть развилка — **задать вопросы списком**, не начинать кодить наугад.
3. После ответов — расписать декомпозицию Фазы 0 в виде ToDo, согласовать со мной.
4. После «ок» — поднимаем монорепо и идём по списку.
